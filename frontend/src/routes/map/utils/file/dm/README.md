# DM（数値地形図データ）→ GeoJSON 変換

## 概要

`index.ts`は、国土交通省の「公共測量標準図式　数値地形図データファイル仕様」に基づくDMファイルをGeoJSONに変換するライブラリです。**旧DM**と**拡張DM**の両フォーマットに対応しています。

出力される GeoJSON の座標は**平面直角座標系（メートル単位）**です。WGS84 経緯度への変換は呼び出し側（`transformGeoJSONParallel` 等）で行います。

## DM ファイルの基本仕様

- **固定長 84 バイト/レコード**（Shift-JIS エンコーディング、改行 CR+LF）
- **座標系**: 平面直角座標系（X = 北方向、Y = 東方向）
- **系番号**: 1〜19（日本全国を19の系に分割、測量法施行令 第2条）

## 旧DM と拡張DM の違い

| 項目 | 旧DM | 拡張DM |
|------|------|--------|
| インデックス | `I ` レコードに座標系番号 | I レコードなし |
| 要素レコード | `F ` + pos17 にデータタイプ | `E` + データタイプ1桁（`E1`〜`E7`） |
| 座標レコード | `D2` / `D3` プレフィックス付き | プレフィックスなし（スペースまたは数字で開始） |
| 座標フィールド幅 | 10バイト × 最大4ペア/行 | 7バイト × 最大6ペア/行 |
| 座標単位 | mapLevel に依存（mm/cm/m） | 図郭座標行の座標値単位フィールドに依存（mm/cm/m） |
| 座標変換 | 生値 → `toMeters()` → 絶対座標 | 図郭原点(m) + 座標値 / coordUnitDivisor |

### フォーマット自動判定

ファイル内に `E1`〜`E7` で始まるレコードがあれば拡張DM、`F ` や `I ` レコードがあれば旧DMと判定します。

## レコード構成

### 共通レコード

| レコード | 先頭 | 内容 |
|----------|------|------|
| 図郭 (a) | `M ` | 図郭識別番号、名称、地図情報レベル |
| 図郭 (b) | 空白開始 | 左下/右上の図郭座標（M直後の行） |
| グループヘッダ | `H ` | 分類コード、要素数 |

### 旧DM 固有レコード

| レコード | 先頭 | 内容 |
|----------|------|------|
| インデックス | `I ` | 座標系番号、計画機関名 |
| 要素 | `F ` | 分類コード、データタイプ、座標数 |
| 2D座標 | `D2` | X,Y ペア（10桁 × 最大4組） |
| 3D座標 | `D3` | X,Y,Z（10桁 × 最大2組） |
| 注記 | `L ` | テキスト、座標、角度、文字高さ |

### 拡張DM 固有レコード

#### E レコード（要素レコード）

```
位置   長さ  内容
[1]    1    "E" 固定
[2]    1    データタイプ (1=面, 2=線, 3=円, 4=円弧, 5=点, 6=方向, 7=注記)
[3-6]  4    分類コード
[7]    1    空白
[8]    1    実データ区分
[9-12] 4    間断区分
[13-16] 4   要素識別番号
[17]   1    精度区分
[18]   1    予備
[19-23] 5   地図情報レベルコード
[24-27] 4   フラグ等
[28-31] 4   座標数
[32-35] 4   座標レコード数
[36-42] 7   X座標（点・注記で埋込時）
[43-49] 7   Y座標（点・注記で埋込時）
```

**例**: `E22101         1 2 02360 00   7   2`
- `E` = 要素レコード
- `2` = 線データ
- `2101` = 分類コード（真幅道路）
- 要素ID = 1、座標数 = 7、レコード数 = 2

#### 座標レコード（E レコードの次行）

```
7バイト × 12値 = 6座標ペア (X, Y) / 行
位置    内容
[1-7]   X1
[8-14]  Y1
[15-21] X2
[22-28] Y2
  ...
[71-77] X6
[78-84] Y6
```

座標値 `0` はパディング（無効値）。座標値の単位は図郭座標行の座標値単位フィールドにより決まります（下記参照）。

**例**: ` 145112  35078 145318  34867 145432  34685`（座標値単位=10(cm)の場合）
- ペア1: X=1451.12m, Y=350.78m（図郭原点からの相対値）

## 座標変換の流れ

### 旧DM

```
生の座標値
  → toMeters(value, mapLevel)  // mm/cm/m → メートル
  → [Easting(m), Northing(m)]  // 平面直角座標
```

### 拡張DM

```
生の座標値
  → 図郭原点(m) + 座標値 / coordUnitDivisor  // 絶対座標（メートル）
  → [Easting(m), Northing(m)]                 // 平面直角座標
```

出力後、呼び出し側で `transformGeoJSONParallel(geojson, projContext)` を使い WGS84 経緯度に変換します。

### 図郭原点と座標値単位の取得

M レコード直後の行（図郭座標行）から左下座標と座標値単位を取得します。
拡張DMの付属行には旧DMの `MB` プレフィックスがないため、フィールド位置が異なります。

#### 旧DM（`MB` レコード）

```
位置     長さ  内容
[1-2]    2    "MB" 固定
[3-9]    7    左下X (m)
[10-16]  7    左下Y (m)
[17-19]  3    座標値単位
[20-26]  7    要素数
[27-32]  6    レコード数
[33-39]  7    右上X (m)
[40-46]  7    右上Y (m)
```

#### 拡張DM（M レコード直後の付属行）

```
位置     長さ  内容
[1-7]    7    左下X (m)
[8-14]   7    左下Y (m)
[15-21]  7    右上X (m)
[22-28]  7    右上Y (m)
  ...          (要素数、レコード数 等)
[45-47]  3    座標値単位
```

> **注意**: 拡張DMでは左下座標の直後に右上座標が続き、座標値単位は [45-47] に位置します。旧DMの `MB` レコードとはフィールド配置が異なります。

**座標値単位と除算器の対応**:

| 座標値単位 | 単位 | 除算器(coordUnitDivisor) | 主な地図情報レベル |
|------------|------|--------------------------|--------------------|
| 1          | mm   | 1000                     | 500, 1000          |
| 10         | cm   | 100                      | 2500, 5000         |
| 1000       | m    | 1                        | 10000              |

座標値単位が取得できない場合は、`mapLevel` から推定するフォールバックが適用されます。

**例**: `"-222900 -32000-221150 -29500     4945  15916 10..."`

- 左下X=-222900m、左下Y=-32000m
- 右上X=-221150m、右上Y=-29500m
- 座標値単位=10(cm) → coordUnitDivisor=100
- 図郭範囲: X方向1750m、Y方向2500m

## 分類コード

4桁コード = レイヤ番号(2桁) + データ項目(2桁)。公共測量標準図式の分類コード表に準拠。

| コード範囲 | 分類 | 主な項目 |
|-----------|------|---------|
| 11xx | 境界・所属界 | 都府県界, 郡市界, 町村界, 大字界 |
| 21xx | 道路 | 真幅道路, 徒歩道, 庭園路 |
| 22xx | 道路施設 | 道路橋, 歩道, 石段, トンネル |
| 23xx | 鉄道 | 普通鉄道, 路面電車, 特殊鉄道 |
| 24xx | 鉄道施設 | 鉄道橋, 跨線橋, 停留所 |
| 30xx | 建物 | 普通建物, 堅ろう建物, 無壁舎 |
| 34xx-35xx | 建物付属物・記号 | 門, 官公署, 学校, 病院, 工場 等 |
| 42xx | 小物体 | 墓碑, 鳥居, タンク, 煙突, 灯台, 送電線 等 |
| 51xx | 水部 | 水がい線, 一条河川, かれ川, 用水路, 湖池 |
| 52xx | 水部構造物 | 桟橋, 滝, せき, 水門 |
| 61xx | 人工斜面・構囲 | 人工斜面, 土堤, かき, へい |
| 62xx | 諸地・場地 | 駐車場, 墓地, 太陽光発電設備 |
| 63xx | 植生・耕地 | 田, 畑, 茶畑, 果樹園, 広葉樹林, 竹林 等 |
| 71xx | 等高線 | 計曲線, 主曲線, 補助曲線, 凹地 |
| 72xx | 変形地 | 土がけ, 岩がけ, 露岩 |
| 73xx | 基準点 | 三角点, 水準点, 多角点, 電子基準点, 標高点 |
| 81xx | 注記 | 地名, 道路名, 建物名, 標高注記 等 |

## 使用方法

```typescript
import { convertDMFileToGeoJSON } from './dm';
import { transformGeoJSONParallel } from '$routes/map/utils/proj';
import { getProjContext, type EpsgCode } from '$routes/map/utils/proj/dict';

// 1. DM → GeoJSON（平面直角座標系のメートル座標）
const dmResult = await convertDMFileToGeoJSON(file, {
  zoneNumber: 3,  // 平面直角座標系の系番号
});

// 2. 平面直角座標 → WGS84 経緯度
const epsgCode = String(dmResult.properties?.epsgCode ?? 6668 + zoneNumber);
const projContext = getProjContext(epsgCode as EpsgCode);
const geojson = await transformGeoJSONParallel(dmResult, projContext);
```

### ConvertOptions

| オプション | 型 | デフォルト | 説明 |
|------------|-----|-----------|------|
| `zoneNumber` | `number` | I レコードから取得、なければ拡張DM: 2, 旧DM: 9 | 平面直角座標系の系番号 (1〜19) |
| `includeAnnotations` | `boolean` | `true` | 注記フィーチャの出力 |
| `coordinatePrecision` | `number` | `8` | 座標の丸め精度 |

### 系番号の事前取得

```typescript
import { getDMZoneInfo, ZONE_REGIONS } from './dm';

const info = await getDMZoneInfo(file);
// info.indexZone: INDEXレコードの系番号（存在する場合）
// info.defaultZone: デフォルトの系番号
// info.drawingName: 図郭名称
// ZONE_REGIONS[3] → '山口・島根・広島'
```
