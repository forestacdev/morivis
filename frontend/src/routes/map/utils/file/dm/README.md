# DM（数値地形図データ）→ GeoJSON 変換

## 概要

`dm.ts` は、国土交通省の「公共測量標準図式 数値地形図データファイル仕様」に基づく DM ファイルを GeoJSON に変換するライブラリです。**旧DM** と **拡張DM** の両フォーマットに対応しています。

## DM ファイルの基本仕様

- **固定長 84 バイト/レコード**（Shift-JIS エンコーディング、改行 CR+LF）
- **座標系**: 平面直角座標系（X = 北方向、Y = 東方向）→ WGS84 経緯度に変換
- **系番号**: 1〜19（日本全国を19の系に分割、測量法施行令 第2条）

## 旧DM と拡張DM の違い

| 項目 | 旧DM | 拡張DM |
|------|------|--------|
| インデックス | `I ` レコードに座標系番号 | I レコードなし |
| 要素レコード | `F ` + pos17 にデータタイプ | `E` + データタイプ1桁（`E1`〜`E7`） |
| 座標レコード | `D2` / `D3` プレフィックス付き | プレフィックスなし（スペースまたは数字で開始） |
| 座標フィールド幅 | 10バイト × 最大4ペア/行 | 7バイト × 最大6ペア/行 |
| 座標単位 | mapLevel に依存（mm/cm/m） | **常にmm**（図郭原点からの相対値） |
| 座標変換 | 生値 → `toMeters()` → 絶対座標 | 図郭原点(m) + 座標値(mm) / 1000 |

### フォーマット自動判定

ファイル内に `E1`〜`E7` で始まるレコードがあれば拡張DM、`F ` や `I ` レコードがあれば旧DMと判定します。

## レコード構成

### 共通レコード

| レコード | 先頭 | 内容 |
|----------|------|------|
| 図郭 (a) | `M ` | 図郭識別番号、名称、地図情報レベル |
| 図郭 (b) | 空白開始 | 左下/右上の図郭座標（M直後の行） |
| グループヘッダ | `H ` | 分類コード、要素数 |

### 旧DM 固有レコード

| レコード | 先頭 | 内容 |
|----------|------|------|
| インデックス | `I ` | 座標系番号、計画機関名 |
| 要素 | `F ` | 分類コード、データタイプ、座標数 |
| 2D座標 | `D2` | X,Y ペア（10桁 × 最大4組） |
| 3D座標 | `D3` | X,Y,Z（10桁 × 最大2組） |
| 注記 | `L ` | テキスト、座標、角度、文字高さ |

### 拡張DM 固有レコード

#### E レコード（要素レコード）

```
位置   長さ  内容
[1]    1    "E" 固定
[2]    1    データタイプ (1=面, 2=線, 3=円, 4=円弧, 5=点, 6=方向, 7=注記)
[3-6]  4    分類コード
[7]    1    空白
[8]    1    実データ区分
[9-12] 4    間断区分
[13-16] 4   要素識別番号
[17]   1    精度区分
[18]   1    予備
[19-23] 5   地図情報レベルコード
[24-27] 4   フラグ等
[28-31] 4   座標数
[32-35] 4   座標レコード数
[36-42] 7   X座標（点・注記で埋込時）
[43-49] 7   Y座標（点・注記で埋込時）
```

**例**: `E22101 0   0   1 2 02380 01   2   1`
- `E` = 要素レコード
- `2` = 線データ
- `2101` = 分類コード（都府県界）
- 要素ID = 1、座標数 = 2、レコード数 = 1

#### 座標レコード（E レコードの次行）

```
7バイト × 12値 = 6座標ペア (X, Y) / 行
位置    内容
[1-7]   X1
[8-14]  Y1
[15-21] X2
[22-28] Y2
  ...
[71-77] X6
[78-84] Y6
```

座標値 `0` はパディング（無効値）。

**例**: ` 245045 232912 251716 271023      0      0`
- ペア1: X=245045mm, Y=232912mm
- ペア2: X=251716mm, Y=271023mm

## 座標変換の流れ

### 旧DM

```
生の座標値
  → toMeters(value, mapLevel)  // mm/cm/m → メートル
  → jpRectToWgs84(x_m, y_m, zone)  // 平面直角 → WGS84
  → [lon, lat]
```

### 拡張DM

```
生の座標値 (mm)
  → 図郭原点X(m) + 座標値 / 1000  // 絶対座標（メートル）
  → 図郭原点Y(m) + 座標値 / 1000
  → jpRectToWgs84(abs_x, abs_y, zone)  // 平面直角 → WGS84
  → [lon, lat]
```

### 図郭原点の取得

M レコード直後の行（図郭座標行）から左下座標を取得します:

```
行例: "  90000      0  96000   8000    22291  40404 ..."
       ↑左下X(m)  ↑左下Y(m) ↑右上X(m) ↑右上Y(m)
```

この例では図郭原点 = (X: 90000m, Y: 0m)、図郭範囲 6000m × 8000m。

## 平面直角座標系 → WGS84 変換

TM（横メルカトル）逆変換を使用。Bowring/Snyder 方式で GRS80 楕円体パラメータに基づきます。

## 使用方法

```typescript
import { convertDMFileToGeoJSON } from './dm';

// ブラウザの File API から直接変換
const geojson = await convertDMFileToGeoJSON(file, {
  zoneNumber: 2,  // 平面直角座標系の系番号（省略時: 拡張DM=2, 旧DM=9）
});

// テキストから変換
import { convertDMtoGeoJSON, decodeShiftJIS } from './dm';
const buffer = await file.arrayBuffer();
const text = decodeShiftJIS(buffer);
const geojson = convertDMtoGeoJSON(text, {
  zoneNumber: 2,
  includeAnnotations: true,   // 注記をフィーチャとして含めるか（default: true）
  coordinatePrecision: 8,     // 座標の小数点以下桁数（default: 8）
});
```

### ConvertOptions

| オプション | 型 | デフォルト | 説明 |
|------------|-----|-----------|------|
| `zoneNumber` | `number` | 拡張DM: 2, 旧DM: 9 | 平面直角座標系の系番号 (1〜19) |
| `coordTransformer` | `CoordTransformer` | 内蔵変換 | カスタム座標変換関数（proj4js等） |
| `includeAnnotations` | `boolean` | `true` | 注記フィーチャの出力 |
| `coordinatePrecision` | `number` | `8` | 座標の丸め精度 |
